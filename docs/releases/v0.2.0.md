# LocalScribe 0.2.0

LocalScribe 0.2.0 turns the project into a more complete live transcription workspace for macOS: better live-session UX, managed WhisperKit model handling, on-device cleanup choices, and stronger transcript editing flow.

## Highlights

- Refined live control room UI with a tighter `Capture Live` workflow and session naming from the main panel
- Live segment editor with inline transcript edits that persist while new chunks keep arriving
- Speaker-aware segmentation, renameable speaker roster, and speaker change markers
- Managed WhisperKit model installation, switching, and install-progress feedback
- Always-on context stitching plus local cleanup via Ollama or MLX
- Safari-focused browser flow plus optional native macOS system-audio helper

## Included in this release

- MLX cleanup runtime support with clearer readiness messaging
- Task-based cleanup labels in the UI instead of backend-centric names
- More explicit live status and input-warning states when capture is active
- Compact capture card layout with the current session status moved next to the live transcript editor
- Updated local install guidance and release branding assets

## Notes

- Apple Silicon remains the preferred path because WhisperKit is the default local inference engine
- The published app is still a local-first developer release, not a notarized `.app` bundle
- For the best first-run experience, start with `tiny` to verify the pipeline and then switch to `large-v3-turbo`
