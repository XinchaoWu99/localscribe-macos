# LocalScribe 0.2.3

## Highlights

- Adds explicit Microsoft Edge browser support with a dedicated Playwright smoke test that starts live microphone capture and verifies transcript text appears.
- Refreshes the browser-facing setup and troubleshooting guidance so Edge is treated as a supported live-microphone path and the preferred browser path for tab audio capture.
- Makes live transcript editing work during capture, including locking a manually edited live caption so later speech continues in the next segment.
- Improves live responsiveness by shortening the startup chunk, skipping unnecessary live cleanup passes on draft-only updates, and reducing per-chunk processing overhead.
- Replaces the deprecated `ScriptProcessorNode` live capture path with `AudioWorkletNode` in modern browsers.

## Notes

- Edge support is covered by `npm run test:e2e:edge` and is also exercised by the default `npm run test:e2e` suite when Microsoft Edge is installed at `/Applications/Microsoft Edge.app`.
- Live transcription still uses chunk-based request/response processing rather than true token streaming, so this release reduces avoidable latency but does not yet provide word-by-word streaming output.
